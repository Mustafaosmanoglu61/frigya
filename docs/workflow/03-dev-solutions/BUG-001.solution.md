## 🛠️ Yaklaşım (yön — kodu AI tool üretsin)

- Server-side `totals` (`islemler.py:75-85`) **olduğu gibi kalsın** → ilk render için doğru.
- Üzerine **client-side recalculation** katmanı: `filterTable()` her çalıştığında görünür (`display !== 'none'`) satırları topla, KPI + tfoot DOM'unu güncelle.
- Toplama için sayıyı satır/hücre `data-*` attribute'undan oku (locale/`$`/virgül parse etme).
- Başarı % = (kâr satırı / görünür satır) × 100; görünür satır 0 ise `'—'`.

**Pseudo akış:**
```
filterTable():
   ...mevcut görünürlük mantığı...
   _recalcTotals()        # ← yeni satır

_recalcTotals():
   visible = görünür <tr>'ler
   proceeds = Σ row.querySelector('.col-proceeds').dataset.raw
   cost     = Σ row.col-cost.dataset.raw
   profit   = Σ max(0, row.dataset.pnl)
   loss     = Σ |min(0, row.dataset.pnl)|
   wins     = count(row.dataset.pnl >= 0)
   net      = proceeds - cost
   → [data-kpi=...] ve [data-toplam=...] hücrelerini yaz
   → görünür 0 ise: KPI '$0.00', Başarı '—', tfoot gizle
```

## ⚠️ Dikkat Et
- `_recalcTotals()` hem sembol filtresinde hem **tarih filtresinde** hem **temizlemede** tetiklenmeli (üçü de `filterTable()`'dan geçiyor → tek noktadan çözülür).
- tfoot zaten `534-536`'da 0 satırda gizleniyor → onu koru, sadece değer güncellemesini ekle.
- Sayı formatı mevcut helper'la tutarlı olsun (`$` + binlik ayraç). Projede zaten format helper'ı var, yenisini yazma — ara/yeniden kullan.
- Para birimi parse riskine girme → `data-raw` ham float tut.
